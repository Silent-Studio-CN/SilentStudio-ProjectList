## Security Policy

### Our Commitment

SilentStudio prioritizes the security of user data and project stability. We are committed to timely responding to security vulnerabilities, handling them transparently, and working with the community (GitHub) to maintain a secure ecosystem.

### Supported Versions

Only the following versions receive security updates:

| Version Category | Security Update Support | Support Cycle |
|------------------|------------------------|---------------|
| Latest Stable | ✅ Full support (including all bug fixes) | Until the next major release |
| Previous Stable | ⚠️ Only critical vulnerabilities are fixed | 6 months after release |
| Old Version (≤2 major versions prior) | 🤚 Fix on request (please report the issue, we will handle it) | Further notice (can be understood as long-term support) |

> Critical vulnerabilities include remote code execution, data leakage, authentication bypass, and other issues that may cause severe consequences.

### Reporting a Vulnerability

**How to Submit**

Report vulnerabilities via the dedicated email **SilentStudio@feedback.email.cn**. Do not disclose vulnerability details in public channels (e.g., Issues, discussions).

**Required Information**

- Vulnerability type and impact scope
- Reproduction steps (including environment configuration)
- Proof of vulnerability (e.g., PoC code, screenshots)
- Your contact information (for follow-up)

> If you cannot provide precise feedback, please describe the problem in detail. We will be grateful. 🙏

### Handling Process

| Stage | Timeline |
|-------|----------|
| Acknowledgment | Reply within 24 hours to confirm receipt |
| Validation | Complete validation and severity assessment within 48 hours |
| Critical Fix | Fix plan within 7 days |
| Medium/Low Fix | Fix plan within 14 days |
| Disclosure | Publish details after the fix is released (credit reporters with consent) |

### Responsible Disclosure Principles

Good-faith reporters will not be held liable for testing, provided they:

- Do not abuse vulnerabilities to access data or damage systems
- Keep information confidential until the vulnerability is fixed

**Last Updated:** 2025-07-11

This policy applies to all projects developed by SilentStudio that offer updates or support.
