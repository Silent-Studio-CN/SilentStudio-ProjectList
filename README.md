## 安全政策

### 我们的承诺

SilentStudio 始终将用户数据安全与项目稳定性放在首位。我们承诺对所有安全漏洞进行及时响应、透明处理，并与社区（GitHub）共同维护安全生态。

### 支持的版本

仅以下版本会接收安全更新：

| 版本类别 | 安全更新支持 | 支持周期 |
|----------|-------------|----------|
| 最新稳定版 | ✅ 全面支持（含所有漏洞修复） | 直至下一个大版本发布 |
| 上一个稳定版 | ⚠️ 仅修复 Critical 漏洞 | 发布后 6 个月 |
| 旧版本（≤2个大版本前）| 🤚 按需修复（请反馈问题，我们将针对处理）| 另行通知（可理解为长期支持）|

> Critical 漏洞包括远程代码执行、数据泄露、身份认证绕过等可能导致严重后果的问题。

### 报告漏洞

**如何提交**

请通过专用邮箱 **SilentStudio@feedback.email.cn** 报告漏洞，请勿在公开渠道（如 Issues、讨论区）发布漏洞细节。

**报告需包含**

- 漏洞类型及影响范围
- 复现步骤（含环境配置）
- 漏洞证明（如 PoC 代码、截图）
- 你的联系方式（用于跟进）

> 如果无法精准反馈，详细描述问题即可，感激不尽 🙏

### 处理流程

| 阶段 | 时限 |
|------|------|
| 确认接收 | 24 小时内回复确认 |
| 漏洞验证 | 48 小时内完成验证并评估严重程度 |
| Critical 修复 | 7 天内提供修复方案 |
| 中低危修复 | 14 天内提供修复计划 |
| 公开披露 | 修复发布后同步公开（经报告者同意可致谢）|

### 负责任披露原则

善意报告者不会因测试行为被追责，但需遵守：

- 不滥用漏洞获取数据或破坏系统
- 漏洞修复前保持信息机密

**最后更新：** 2025-07-11

本政策适用于 SilentStudio 开发的所有具备更新功能或提供支持的项目。
